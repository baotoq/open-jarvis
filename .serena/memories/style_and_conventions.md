# Style and Conventions

## Go
- Follow standard Go conventions (gofmt, go vet)
- go-zero framework patterns for API/RPC services
- Naming: camelCase for unexported, PascalCase for exported symbols
- Error handling: explicit error returns (no panics in library code)

## TypeScript / Next.js
- Strict TypeScript (`"strict": true` in tsconfig)
- Next.js App Router conventions
- npm as package manager
- Naming: camelCase for variables/functions, PascalCase for components/types

## General
- Linting/formatting tools not yet chosen — to be decided as project grows
