<objective>
Add four conversation management HTTP endpoints (list, get, delete, get-messages) and update `StreamChat` to assign UUIDs, persist conversation titles, and emit a final SSE done-event carrying the session ID.

Purpose: CHAT-03 and UI-02 both depend on backend endpoints that expose stored conversations. The messages endpoint is specifically required by the frontend to load prior messages when a user clicks a sidebar conversation entry. This plan wires the persistence layer from Plan 01 to the HTTP API and fixes the SSE session ID protocol gap.
Output: Four new go-zero handler/logic pairs, updated chatlogic with UUID + title + done event, routes registered in main.go, tests for all new behaviors.
</objective>

    ```go
    func (l *GetConvMessagesLogic) GetConversationMessages(id string) ([]types.MessageResponse, error) {
        msgs := l.svcCtx.Store.Get(id)  // returns nil/empty slice if conversation not found
        result := make([]types.MessageResponse, 0, len(msgs))
        for _, m := range msgs {
            content, _ := m.Content.(string)  // openai.ChatCompletionMessage.Content is interface{}
            result = append(result, types.MessageResponse{Role: string(m.Role), Content: content})
        }
        return result, nil
    }
    ```
