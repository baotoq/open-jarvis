**1. Tiltfile** (at project root — no extension, Starlark syntax):

```python
# Prerequisite guard: fail fast if air is not installed
local('which air || (echo "ERROR: air not installed. Run: go install github.com/air-verse/air@latest" && exit 1)', quiet=True)

local_resource(
    'backend',
    serve_cmd='air',
    serve_dir='src/backend',
    deps=['src/backend'],
    ignore=['src/backend/tmp', 'src/backend/data'],
    readiness_probe=probe(
        tcp_socket=tcp_socket_action(port=8888),
        period_secs=2,
        failure_threshold=15,
    ),
    links=[link('http://localhost:8888', 'Backend API')],
    labels=['services'],
)

local_resource(
    'frontend',
    serve_cmd='npm run dev',
    serve_dir='src/frontend',
    deps=['src/frontend'],
    ignore=['src/frontend/.next', 'src/frontend/node_modules'],
    resource_deps=['backend'],
    links=[link('http://localhost:3000', 'Frontend')],
    labels=['services'],
)

local_resource(
    'open-browser',
    cmd='open http://localhost:3000',
    resource_deps=['frontend'],
    auto_init=True,
    labels=['dev'],
)
```

Notes on the Tiltfile:
- `serve_cmd` (not `cmd`) is mandatory for long-running processes — `cmd` exits immediately
- `serve_dir` sets working directory per resource — air needs src/backend, npm needs src/frontend
- `ignore` for `tmp/` and `data/` prevents Tilt thrashing on air build output and SQLite writes
- `readiness_probe` with TCP on port 8888 ensures frontend resource_deps waits for backend to actually listen
- `open-browser` uses macOS `open`; Linux users should change to `xdg-open` (noted in README)
- `auto_init=True` runs open-browser once on startup (not on restart)

**2. src/backend/.air.toml**:

```toml
root = "."
tmp_dir = "tmp"

[build]
cmd = "go build -o ./tmp/main ./cmd/main.go"
bin = "tmp/main"
include_ext = ["go"]
exclude_dir = ["tmp", "data"]
exclude_regex = ["_test\\.go"]
exclude_unchanged = true
delay = 500
stop_on_error = true
send_interrupt = true
kill_delay = 500000000

[misc]
clean_on_exit = true

[screen]
clear_on_rebuild = false
keep_scroll = true
```

Notes on .air.toml:
- `root = "."` — air runs from src/backend/ (set by Tiltfile serve_dir), so `.` is src/backend/
- `exclude_regex = ["_test\\.go"]` — test files excluded; air rebuilds production binary only
- `exclude_unchanged = true` — skips rebuild if no Go files changed (performance)
- `stop_on_error = true` — air surfaces build errors in Tilt log immediately
- `send_interrupt = true` + `kill_delay` — graceful shutdown for go-zero server
- Do NOT add `[proxy]` section — air proxy not needed (Tilt orchestrates everything)

tilt down
