  <name>Task 1: Embed frontend static export into Go binary and serve</name>
  <files>src/cmd/main.go, frontend/next.config.ts</files>
  <action>
    The Go binary must embed the Next.js static export and serve it alongside the API.

    Step 1 — Direct Next.js output into the Go module tree.
    go:embed paths must be under the module root (where go.mod lives). Since go.mod is at src/, update frontend/next.config.ts to write the build output directly into src/frontend/out:
    ```ts
    const nextConfig = {
      output: 'export',
      distDir: '../src/frontend/out',
    }
    export default nextConfig
    ```
    Then build the frontend:
    cd /Users/baotoq/Work/open-jarvis/frontend && npm run build
    Verify src/frontend/out/ now exists.

    Step 2 — Update src/cmd/main.go.
    Add at the top of the file (package-level, before main):
    ```go
    //go:embed frontend/out
    var frontendFS embed.FS
    ```
    Add required imports: "embed", "io/fs", "net/http", "log"

    In main(), after registering the /api/chat/stream SSE route, add the static file server.
    Use fs.Sub to strip the "frontend/out" prefix, then wrap with http.FileServer.
    Register a catch-all GET route AFTER all API routes so /api/* routes take priority:
    ```go
    subFS, err := fs.Sub(frontendFS, "frontend/out")
    if err != nil {
        log.Fatal(err)
    }
    fileHandler := http.FileServer(http.FS(subFS))
    server.AddRoute(rest.Route{
        Method:  http.MethodGet,
        Path:    "/*",
        Handler: fileHandler.ServeHTTP,
    })
    ```

    Step 3 — Build and verify:
    cd /Users/baotoq/Work/open-jarvis/src && go build ./...

    Note: The //go:embed directive is evaluated at compile time. The frontend/out directory must exist in src/ before running go build. The correct build order is always: npm run build (in frontend/) then go build (in src/).
  </action>
