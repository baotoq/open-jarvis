<task type="auto">
  <name>Task 2: Write .golangci.yml and update CLAUDE.md</name>
  <files>src/backend/.golangci.yml, CLAUDE.md</files>
  <action>
Create src/backend/.golangci.yml with the golangci-lint v2 format. CRITICAL: use v2 schema keys — `linters.settings` (not `linters-settings`), `linters.exclusions.rules` (not `issues.exclude-rules`), `linters.default: none` (not `linters.disable-all: true`), top-level `version: "2"`.

Content for src/backend/.golangci.yml:
```yaml
version: "2"

linters:
  default: none
  enable:
    - errcheck
    - ineffassign
    - staticcheck
    - revive
    - unused

  settings:
    revive:
      rules:
        - name: blank-imports
        - name: context-as-argument
        - name: error-return
        - name: error-strings
        - name: exported
        - name: var-naming
        - name: package-comments
        - name: unused-parameter

  exclusions:
    rules:
      # go-zero uses json struct tag extensions (default=, optional) that are
      # unknown to staticcheck SA5008 — these are intentional and correct
      - linters: [staticcheck]
        text: "SA5008"
      # SQLite driver must be registered via blank import; this is intentional
      - linters: [revive]
        text: "blank-imports"
        path: "svc/servicecontext.go"

formatters:
  enable:
    - goimports

  settings:
    goimports:
      local-prefixes:
        - open-jarvis

run:
  timeout: 5m
```

Then update CLAUDE.md (root of repo). In the `### Go (Backend)` commands section, add `golangci-lint run ./...` as a lint command alongside `go vet ./...`. The line should read:
```
golangci-lint run ./...  # lint (run from src/backend/)
```

After writing both files, verify the config is parseable:
```
cd src/backend && golangci-lint config verify
```

Then run the linter to confirm it starts and processes the codebase (issues are expected and acceptable at this stage):
```
cd src/backend && golangci-lint run ./... || true
```
